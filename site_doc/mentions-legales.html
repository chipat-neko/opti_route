Mentions legales & vie privee.

Mise a jour : 13 mai 2026. Cette page resume la politique de confidentialite et les conditions d'utilisation de l'application opti_route et de son site compagnon. Elle vaut pour la version actuelle (Phase 1 — local, gratuit, sans backend).

1. Editeur

L'application et ce site sont edites par CALOTE Noah, chauffeur-livreur independant base en France, en sa qualite d'auteur personnel. Le projet est open-source (code public sur GitHub) et n'est rattache a aucune societe commerciale en Phase 1.

2. Hebergement

  • Site web : heberge par GitHub Pages (GitHub Inc., San Francisco, USA).
  • Application Android : installee localement sur le telephone de l'utilisateur. Aucun serveur opti_route n'heberge de donnees utilisateur.

3. Donnees personnelles (RGPD)

3.1. Ce qui reste sur ton telephone

Toutes les donnees que tu saisies dans l'app sont stockees localement dans une base SQLite (via Drift) :

  • Tournees, arrets, statuts (livre/echec), raisons d'echec
  • Carnet d'adresses : nom client, adresse, GPS, photos eventuelles
  • Coequipiers : nom, telephone, couleur
  • Photos preuves de livraison (repertoire prive de l'app)
  • Profil entreprise : nom, SIRET, slogan (si renseigne)
  • Position GPS au moment du "Marquer livre / echec" (preuve de passage)

Rien de tout cela n'est uploade sur un serveur tiers. Les donnees sortent uniquement quand TU les exportes explicitement (PDF, CSV, vCard, partage WhatsApp/SMS, etc.).

3.2. APIs externes utilisees

Pour le geocodage et l'optimisation, l'app interroge ces APIs publiques :

  • Base Adresse Nationale (BAN) — api-adresse.data.gouv.fr. Service public francais (IGN / La Poste / DGFiP). Aucune cle. Envoie : adresse texte tapee. Recoit : coordonnees + libelle officiel.
  • Recherche d'Entreprises — recherche-entreprises.api.gouv.fr. Service public francais (data.gouv.fr / INSEE). Aucune cle. Envoie : nom entreprise. Recoit : SIREN + adresse siege.
  • Photon — photon.komoot.io. Geocoder OSS base sur OpenStreetMap, maintenu par Komoot (Allemagne). Envoie : nom enseigne. Recoit : coordonnees + categorie OSM.
  • OpenRouteService (ORS) — api.openrouteservice.org. Service de routing open-source (HeiGIT, Allemagne). Cle API requise (gratuite, 500 optimisations/jour, sans CB). Envoie : liste de coordonnees a optimiser. Recoit : ordre optimal + duree + distance + trace.

Aucune donnee personnelle identifiable n'est envoyee a ces APIs : seulement des adresses et coordonnees.

3.3. Permissions Android demandees

  • INTERNET : appels aux APIs de geocodage et ORS
  • ACCESS_FINE_LOCATION : GPS pour le mode tournee en cours et la preuve de passage
  • CAMERA : scan bordereaux + photo preuve
  • POST_NOTIFICATIONS (Android 13+) : rappels de tournee
  • SCHEDULE_EXACT_ALARM : notifs precises a l'heure choisie

3.4. Tes droits RGPD

Comme aucune donnee n'est centralisee, tes droits s'exercent directement :

  • Droit d'acces : tout est dans l'app, tu peux exporter en CSV/vCard a tout moment.
  • Droit a l'effacement : Parametres → Cache → "Reinitialiser la base" supprime tout. Desinstaller l'app efface egalement la base.
  • Droit a la portabilite : exports CSV/vCard standards (RFC 4180, RFC 2426).
  • Droit d'opposition : ne pas installer l'app.

4. Site web compagnon

Le site (cette page incluse) est statique. Il n'utilise :

  • Pas de cookies (saufs ceux strictement techniques de GitHub Pages, hors notre controle)
  • Pas de tracking analytique (Google Analytics, Plausible, etc.)
  • Pas de publicite
  • Pas de formulaire contact qui collecterait des emails

Le mode sombre / palette utilise localStorage pour persister ton choix entre visites. C'est techniquement un stockage local, pas un cookie, et il ne sort jamais de ton navigateur.

Le dashboard CSV (`dashboard.html`) parse ton fichier CSV dans le navigateur, en memoire. Aucun upload serveur. Quand tu fermes l'onglet, les donnees sont perdues.

Les marques tierces citees (Chronopost, Colissimo, etc.) appartiennent a leurs detenteurs respectifs ; leur citation est purement descriptive (formats de bordereaux supportes).

6. Limitation de responsabilite

L'app opti_route est fournie "telle quelle", sans garantie d'aucune sorte. L'utilisateur reste responsable de :

  • La sauvegarde de ses donnees (exports CSV/vCard reguliers)
  • La conformite des donnees clients qu'il y stocke (RGPD si professionnel)
  • Le respect des regles de la route (l'app ne se substitue pas a un GPS legal)
  • La verification des optimisations proposees (l'algorithme peut suggerer une route non praticable selon le vehicule)

7. Contact

Pour toute question, ouvre une issue sur le repo GitHub. Pas d'email de support en Phase 1 (projet personnel).

Resume : aucune donnee ne quitte ton telephone sans ton accord explicite. L'app est un outil pro local, pas un service en ligne.
