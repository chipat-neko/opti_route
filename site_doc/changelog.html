v2.3.0+2016 2026-05-14 Stable
Sauvegardes, modernisation deps, qualite code

Ajoute

  • Sauvegarde complete de l'app dans un .zip portable (DB SQLite + photos preuves + manifest), partageable via Drive / mail / clef USB.
  • Restore depuis un .zip avec deferred swap au prochain boot (evite les conflits Drift open/close).
  • Auto-backup local periodique (jamais / hebdo / mensuel) dans /Android/data/<pkg>/files/auto_backups/, rotation des 5 derniers.
  • Page "Mes backups" : lister, restaurer, partager, supprimer les .zip generes automatiquement.
  • FAB "Backup maintenant" : creer un .zip manuel a la demande sans attendre la prochaine echeance auto.
  • Notification post-backup auto (importance low, respecte le creneau "ne pas deranger").
  • 22 tests E2E couvrant BackupService, AutoBackupService et BackupsListService.

Modifie

  • Modernisation des dependances (7 paquets bumpes) : connectivity_plus 6→7, share_plus 11→12, file_picker 8→11, flutter_local_notifications 18→21, timezone 0.10→0.11, local_auth 2→3, local_auth_android 1→2.
  • Refactor parametres_screen.dart : 1522 → 1107 lignes (-27 %). Sections Donnees et "Ne pas deranger" extraites en fichiers dedies.
  • Suite de tests : 570/570 verts (+8 BackupsListService), 0 issue analyzer.
v1.1.0+2 2026-05-13
Mode terrain, securite & accessibilite web

Ajoute

  • Compteur de colis sur la stat-row + bandeau de progression en haut de la tournee.
  • Re-ordonner manuellement les arrets en drag-and-drop (intra-tournee).
  • Nouvelle icone d'app (pin + route, palette Claude Design) et splash screen cream.
  • Pages site : changelog, install-apk, gallery.
  • Schema.org JSON-LD SoftwareApplication sur la home (SEO rich snippet Google).
  • Dashboard CSV : persistance localStorage, le dernier CSV charge est automatiquement re-affiche au reload de la page.

Modifie

  • APK desormais decoupe par ABI (armeabi-v7a + arm64-v8a) : ~55-65 MB par variant au lieu de 98 MB monolithique.
  • Accessibilite site : :focus-visible global, aria-live sur le dashboard, scope sur les en-tetes de tableau.

Corrige

  • Mojibakes UTF-8 residuels (re-encodage des accents) sur quelques fichiers archives.
v1.1.0 2026-05-10
Vagues A a H : facturation, parsers bordereaux, chef d'equipe

Ajoute

  • Mode chef d'equipe : creation tournees, affectation des arrets aux coequipiers, partage WhatsApp/SMS cible, export PDF par coequipier.
  • Facturation indicative : recap mensuel HT (par arret / par colis / au km), cout carburant estime, marge brute. NB : pas une facture officielle (sans TVA / mentions legales).
  • Parsers bordereaux : MESEXP (format generique), Colissimo / La Poste, Chronopost.
  • 4 palettes : cream (defaut), parchment, mist, dark.
  • Carnet d'adresses avec import/export CSV + vCard pour double sauvegarde dans les Contacts Android.
  • Statistiques : revenus, km, distance moyenne, colis livres, exports CSV.
  • Site web 10 pages : home, features, roadmap, entreprise (avec calculateur ROI), faq, guide-csv, dashboard CSV viewer, mentions legales.

Modifie

  • Schema Drift migre de v17 a v21 (4 migrations en place).
  • 477 tests passants au moment de la release.
v1.0.0 2026-04-09
Premiere version stable — Phase 1 lancee

Ajoute

  • Creation, edition, optimisation de tournees via OpenRouteService (algorithme VROOM).
  • Cascade de geocodage : BAN (Base Adresse Nationale FR) -> SIRENE -> Photon.
  • Marquage livre / echec / a passer + photo de preuve a la livraison.
  • Profils driving-car et driving-hgv (poids lourd).
  • Export PDF des tournees, parametres carburant (L/100km, prix carburant), tarifs.
  • Mode hors-ligne partiel : creation manuelle d'arrets sans connexion, geocodage differe quand reseau revient.
